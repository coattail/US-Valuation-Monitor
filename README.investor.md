## 1. 项目一句话

**US Valuation Monitor** 是一个聚焦美股估值监测的“数据+应用”平台，覆盖指数、行业与头部公司，提供可日更的估值基础设施与多端产品形态（Web / 小程序 / API）。

---

## 2. 市场痛点

- 估值数据分散在多个来源，口径不一致、可比性差。
- 投资者缺少“连续可追踪”的估值时序，而非单点快照。
- 通用行情产品更偏价格跟踪，估值维度深度不足。
- 手工整理估值数据效率低，难以形成可复用系统。

---

## 3. 我们的解决方案

我们构建了一个可持续运行的估值监测系统：

- **数据层**：多源抓取、融合、校验、标准化，支持日更。
- **模型层**：统一估值指标口径（TTM PE / Forward PE / PB / Earnings Yield）与历史分位体系。
- **服务层**：统一 API 输出快照、时序、热力、自选、提醒。
- **产品层**：Web 与微信小程序双端展示，支持指数与公司看板。

---

## 4. 当前产品能力（已上线）

### 4.1 覆盖范围
- 17 个指数/行业监测对象（核心指数 + 标普行业代理）
- Top 100 美股公司估值看板与详情页

### 4.2 核心功能
- 多指标估值跟踪：`pe_ttm` / `pe_forward` / `pb` / `earnings_yield`
- 全历史 / 10Y / 5Y 分位与估值区间识别
- 指数热力与横向比较
- 企业时序估值、分位子图、最新快照
- 自选、提醒、日更任务触发

### 4.3 数据可信性机制（关键）
- 指数估值沿用历史同源链路，避免短期源切换噪声。
- 企业最新估值优先采用 Yahoo 可信来源覆盖。
- 对企业 `Forward PE` 源切换场景使用“价格修正固定系数重基准”，保证与 Yahoo 最新值衔接、同时保留历史可比性。

---

## 5. 项目护城河

- **工程化数据管线**：可稳定日更并自动提交数据变更。
- **跨端统一口径**：同一份标准化数据供 Web/API/小程序使用。
- **估值连续性设计**：不仅“抓到数据”，更解决时序连续与口径切换问题。
- **可扩展架构**：可继续扩展更多市场、更多资产类别与更多策略指标。

---

## 6. 商业化方向（可选路径）

- **B2C 订阅**：高级估值看板、估值提醒、深度对比与历史回溯能力。
- **B2B 数据服务**：向投研团队、媒体、量化开发者提供 API / 数据包。
- **投顾/内容生态合作**：估值专题、行业监测组件、白标化嵌入。
- **海外市场扩展**：从美股估值监测延展至全球主要市场。
